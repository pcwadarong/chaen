## 🚨 주요 고민 및 해결 과정

> 주요 고민이나 문제 해결 과정 공유

### 문제

- 기존 방명록 작성 폼은 위치와 이름이 방명록 전용이라 아티클 댓글에서 그대로 재사용하기 어려웠습니다.
- 아티클 댓글은 한 줄 레이아웃과 심플한 정렬/페이지네이션 UI가 필요했지만, 방명록의 고정 하단 작성 UX는 그대로 유지해야 했습니다.
- 초기 계획에는 비밀글이 포함돼 있었지만, 최신 요구사항에서는 아티클 댓글에서 비밀글을 완전히 제거해야 했습니다.
- DB 스키마가 아직 저장소에 없어서 현재 UI/도메인 기준을 그대로 반영할 마이그레이션 파일이 필요했습니다.

### 해결 과정

- 작성 폼을 `CommentComposeForm`으로 공용화하고 `topRowWrap` 같은 옵션을 추가해 아티클 댓글만 한 줄 배치로 고정했습니다.
- 댓글 위젯은 서버 초기 데이터 + 클라이언트 재조회 구조로 단순화해 정렬/페이지네이션/수정/삭제 후 일관성을 우선했습니다.
- 아티클 댓글 도메인에서 `is_secret`, 마스킹, 비밀글 확인 엔드포인트 의존을 제거하고 일반 댓글 모델로 정리했습니다.
- 마이그레이션에는 자기참조 FK, 길이 제약, public read policy, 조회 인덱스를 포함해 실제 런타임 코드와 맞는 스키마를 정의했습니다.
