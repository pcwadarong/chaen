## 📝 작업 내용

1. 댓글 추가 이후 아티클 상세 페이지가 서버 렌더링 에러로 무너지는 문제를 분석하고 복구 경로를 추가한다.
2. 모바일 아티클 본문 이미지가 뷰포트를 밀어내지 않도록 마크다운 이미지 렌더링을 보정한다.
3. 토스트 컴포넌트 디자인을 모바일/데스크톱 모두에서 읽기 쉬운 카드형 레이아웃으로 개선한다.
4. 수동 확인용 토스트 프리뷰 페이지를 임시로 추가한다.
5. 공통 floating shadow 토큰을 추가하고 주요 떠 있는 UI의 그림자를 같은 값으로 통일한다.

<br/>

## 🚨 주요 고민 및 해결 과정

### 문제

- 댓글 생성 후 캐시 재검증 시 아티클 상세 서버 컴포넌트가 다시 렌더링되며 태그 스키마 의존 에러 가능성이 있었다.
- 마크다운 이미지에 폭 제한이 없어 모바일에서 가로 스크롤이 발생했다.

### 해결 과정

- 태그 라벨 조회가 불가능한 경우에도 상세 페이지는 원본 slug 기반으로 렌더링되도록 복구 정책을 추가한다.
- 마크다운 `img` 노드를 명시적으로 렌더링해 `max-width: 100%` 기준의 반응형 스타일을 보장한다.
- 토스트는 상태 점, 카드 표면, 원형 닫기 버튼을 갖는 구조로 재설계해 가독성과 조작성을 높인다.
- 토스트 시각 확인을 위해 `/{locale}/dev/toast` 임시 프리뷰 페이지를 추가한다.
- 반복되던 패널/토스트/팝오버 그림자를 Panda shadow token으로 통합한다.
