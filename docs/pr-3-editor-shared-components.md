# PR 3 - Editor Shared Components

## 작업 목표

- 에디터 전반에서 재사용할 입력 컴포넌트를 프로젝트 구조에 맞게 공용 레이어로 정리한다.
- 태그 선택, 슬러그 입력, 링크 임베드 카드를 각각 독립적으로 검증 가능한 단위로 나눠 구현한다.

## 핵심 정책

- `src/components` 대신 현재 프로젝트의 FSD 구조에 맞춰 공용 UI는 `src/shared/ui` 아래에 배치한다.
- 상태가 필요한 입력 UI만 `use client`를 사용하고, 마크다운 렌더링 확장은 기존 `src/shared/ui/markdown` 축에서 이어간다.
- 테스트를 먼저 고정하고 기능 단위로 나눠 커밋한다.

## 진행 현황

- 완료: `TagSelector` 구현을 위한 배치 방향 확정
- 진행 중: 검색, 선택/해제, 접기 가능한 `TagSelector` 컴포넌트 및 테스트 추가
- 예정: `SlugInput`, `LinkEmbedCard`, `/api/og` 라우트, markdown embed 렌더링 확장

## 판단 근거

- 기존 코드베이스는 입력 컴포넌트, 마크다운 렌더러, 버튼/아이콘이 모두 `src/shared/ui`에 모여 있어 이 축을 유지하는 편이 재사용성과 탐색성이 높다.
- 링크 임베드는 단일 컴포넌트보다 markdown 렌더링 파이프라인과 API 라우트가 함께 맞물리므로 마지막 커밋 단위로 묶는 것이 회귀 범위를 통제하기 쉽다.
